# Migration guide — v1.196.0 "Mechanical Floor"

**Audience:** operators and downstream tooling upgrading from any v1.18x/v1.19x
release. **TL;DR:** additive and backward-compatible. No config is *required* to
change. The items below are opt-in surfaces and behavior clarifications worth
knowing about; each ends with a backward-compat note.

## 1. `claude-sonnet-5` executor tier (opt-in)

The cheval model catalog gained `claude-sonnet-5` (1M context, $3/$15 per MTok),
and the `advisor_strategy.tier_aliases.executor.anthropic` alias now points at
it. This is **dormant unless you have opted into tiered dispatch** — the
`stage_routing.*` flags remain `false` by default, so no live dispatch changes.

- **To use it:** set `advisor_strategy.stage_routing.<stage>: true` for the
  stage you want on the executor tier (as before — the only change is *which*
  Sonnet the executor resolves to).
- **Backward-compat:** if you pinned `tier_aliases.executor.anthropic` yourself,
  your value wins. The bare `claude-sonnet-5` alias is additive.
- **Note:** the flatline-scorer tier A/B (cycle-120) did **not** flip the scorer
  to the executor tier — that stage stays on advisor. See the report for the
  pre-registered decision.

## 2. `registry.reserved_skill_names` (restored)

`.loa.config.yaml` regained the `registry.reserved_skill_names` list (the 8
golden-path skill slugs). If you install registry/construct packs, a pack skill
whose slug matches one of these is filtered from `list`/`loadable` and blocked
by `preload`.

```yaml
registry:
  reserved_skill_names:
    - discovering-requirements
    - designing-architecture
    - planning-sprints
    - implementing-tasks
    - reviewing-code
    - auditing-security
    - deploying-infrastructure
    - translating-for-executives
```

- **Backward-compat:** if the key is absent, no name is treated as reserved
  (the pre-v1.196 behavior). Add the block to re-enable shadow protection; the
  shipped default already contains it.
- **Downstream tooling:** this key is read by `is_reserved_skill_name`
  (`constructs-lib.sh`) via `yq`; if you author packs, avoid these 8 slugs.

## 3. `calling_primitive` accepts skill/agent slugs (trajectory consumers)

The `model.invoke.complete` and `class.resolved` payload schemas previously
constrained `calling_primitive` to `L1`–`L7` or null. They now **also** accept
skill/agent slugs — the value cheval has actually emitted since cycle-112 D-6
(e.g. `"/review-sprint"`, `"adversarial-review:review"`).

- **If you validate `.run/model-invoke.jsonl` against these schemas:** pull the
  updated schema files (`.claude/data/trajectory-schemas/model-events/*.json`)
  — old copies will reject slug values that production has been emitting.
- **Pattern:** `^/?[A-Za-z][A-Za-z0-9._:/-]{0,127}$`. Malformed values (spaces,
  leading digit) are still rejected.
- **Backward-compat:** every previously-valid value (L1–L7, null) still
  validates. This only *widens* the accepted set.

## 4. `AUTH_REVOKED` in the model-error taxonomy

The `model-error` `error_class` enum grew from 11 to 12 (`AUTH_REVOKED`, the
walkable runtime token-revocation class the cheval emitter has produced since
PR #1101). Same guidance as §3: refresh `model-error.schema.json` if you
validate error envelopes downstream.

## 5. Installer safety (`mount-loa.sh` / `mount-submodule.sh`)

Behavior is stricter and safer; normal installs are unaffected:

- **Missing flag operands now fail loudly.** `mount-loa.sh --branch` (no value)
  previously consumed the *next* flag or an empty string as the branch; it now
  errors with `Option --branch requires a value`. If any wrapper of yours
  relied on the silent-swallow behavior, pass explicit values.
- **Output uses `printf`, not `echo -e`.** Log lines with backslashes or a
  leading `-` in a ref/path render verbatim now.
- **Symlink boundary check tightened.** A symlink resolving to a *sibling* of
  the repo root (e.g. `/x/repo-evil` when root is `/x/repo`) is now correctly
  rejected as escaping bounds.

Enforced by `check-installer-safety.sh` (run it yourself against any forked
installer) + `tests/unit/installer-safety.bats`.

## Tracing / verification knobs

- `bash .claude/scripts/check-installer-safety.sh` — audit the two installers
  (exit 1 on any of the three unsafe patterns).
- `LOA_SKILLS_DIR=<dir>` — redirect where `symlink_pack_skills` links pack
  skills (used by tests; also handy for dry-run installs into a scratch tree).
- `bats --print-output-on-failure tests/unit/` — the CI invocation; failing
  tests now print their captured output locally too.

## Non-goals / explicitly unchanged

- No change to the audit-envelope chain, dispatch defaults (`auto`), or the
  post-merge automation contract.
- The 4 open community PRs with requested changes (#1200, #1167, #1153, #1109)
  are **not** part of this release.
