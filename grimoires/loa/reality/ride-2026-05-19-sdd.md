## 1. Architecture

A config-driven, multi-chain blockchain indexer built on Envio HyperIndex V3.
[GROUNDED: `package.json` `envio@3.0.0-alpha.14`; `config.yaml`]

```
6 EVM chains → HyperSync ingestion → handler dispatch → entity writes → Postgres → GraphQL API
```
[GROUNDED: Envio HyperIndex model; `config.yaml` networks; `src/handlers/`]

The system has three declarative inputs and one code surface:
- `config.yaml` — the indexer manifest (contracts, events, networks, start blocks). [GROUNDED]
- `schema.graphql` — the output entity schema (93 types). [GROUNDED: `schema.graphql`]
- `src/` — handler logic (TypeScript) that transforms events into entities. [GROUNDED: `src/`]
- `generated/` — `envio codegen` output (ReScript runtime + TS bindings); never hand-edited. [GROUNDED: `generated/`, `package.json` `codegen` script]

### Event vs. Stat Entities
Most domains keep an immutable event entity (e.g. `Transfer`, `FatBeraDeposit`)
plus a mutable aggregate-stat entity (`CollectionStat`, `*Stats`) updated in the
same handler. [GROUNDED: `schema.graphql`; `honey-jar-nfts.ts`]

## 6. Key Design Decisions

| Decision | Rationale | Grounding |
|----------|-----------|-----------|
| Config-as-manifest | Contracts/addresses/start-blocks declarative; handlers carry only logic | [GROUNDED: `config.yaml`, `src/handlers/`] |
| Explicit Berachain HyperSync endpoint | Auto-discovery missed Berachain on alpha.14 | [GROUNDED: `config.yaml:661-668` comment] |
| Cross-chain holder aggregation | Bridged NFTs roll up to one canonical collection | [GROUNDED: `handlers/constants.ts` `ADDRESS_TO_COLLECTION`] |
| Universal `Action` entity | Decouples quest verification from per-protocol handlers | [GROUNDED: `src/lib/actions.ts`, `schema.graphql:1`] |
| One module per product | Small blast radius per change; SCALE.md checklist | [GROUNDED: `src/handlers/`, `SCALE.md`] |

## 7. Cross-Cutting Concerns

- **Reindex cost**: `config.yaml` changes can trigger multi-hour reindexes; `SCALE.md`
  is the mandatory operator playbook. [GROUNDED: `README.md`, `SCALE.md`]
- **Deployment redundancy**: two parallel HyperIndex deployments kept in sync.
  [GROUNDED: `DEPLOYMENT_GUIDE.md`]
- **Testing**: only one indexer test (`test/fatbera-core.test.ts`) — coverage is
  thin relative to 84 handlers. [GROUNDED: `test/`]
- **Repo hygiene**: indexer code shares the repo with Loa framework content; see
  `reality/hygiene-report.md`. [GROUNDED: filesystem scan]
