## Gaps Identified

### GOV-1: No indexer release versioning (Medium)
- Only git tag is `loa@v0.6.0` (framework). `package.json` version is `0.1.0` and
  has not moved. The indexer is deployed continuously to HyperIndex (endpoints
  `b5da47c` / `914708e`) — deployment IDs act as the de-facto version, but there is
  no tag/changelog trail tying a deployment to a commit.
- **Recommendation**: Either adopt deployment-ID-as-version explicitly in docs, or
  tag indexer releases separately from framework tags.

### GOV-2: CHANGELOG.md scope mismatch (Medium)
- `CHANGELOG.md` (378 KB) is the Loa framework changelog. A reader looking for
  indexer history finds framework history instead.
- **Recommendation**: Separate indexer changelog, or move framework changelog under
  `.claude/`. See `reality/hygiene-report.md` for the broader framework-mixing issue.

### GOV-3: SECURITY.md version table stale (Low)
- Supported-versions table (`0.1.x` / `0.2.x`) does not correspond to any real
  indexer release line. The disclosure-policy text itself is fine.
- **Recommendation**: Update or remove the version table.

## Healthy Governance

- CODEOWNERS, branch protection, PR template, issue templates, and Dependabot are
  all configured under `.github/`.
- The PR template prompts contributors for SCALE.md source-addition fields — good
  process discipline around the indexer's highest-risk change type (`config.yaml`).
- `SECURITY.md` has a real disclosure policy.

## Overall

Governance scaffolding is **present and mostly solid** at the `.github/` level.
The weaknesses are all downstream of the same root cause: indexer and Loa framework
content share one repo, so versioning and changelog artifacts describe the framework
instead of the indexer.
