The patch violates several core atomicity, rollback, recovery, and revocation invariants. In particular, incomplete generations can become active, recovery approvals are replayable or mutable, and revoked artifacts can return to green state.

- [P1] Validate the complete object closure before CAS — /private/tmp/sonar-agent-dream-outcome/src/truth-contract/filesystem-registry-store.ts:378-380
  When `compareAndSwapRoot` receives an activation whose referenced objects were not persisted, this pointer write still succeeds and `readRoot` exposes the incomplete generation. The new filesystem tests exercise exactly this path without writing the root's objects, violating the invariant that partial generations are unobservable; verify every referenced digest and byte length before activating the pointer.

- [P1] Verify the nested signed root during activation checks — /private/tmp/sonar-agent-dream-outcome/src/truth-contract/trust-control-plane.ts:346-357
  If an activation contains a modified `unsigned_root` with a stale `root_hash` or root signature, this function only recomputes the outer activation hash and verifies the activation signature. A freshly signed activation can therefore authenticate an internally invalid root; call the bundle-root verifier and enforce matching root/activation issuers before returning it as trusted.

- [P1] Bind challenge timestamps into recovery approvals — /private/tmp/sonar-agent-dream-outcome/src/truth-contract/trust-control-plane.ts:243-249
  For a signed recovery request, the approvals bind only environment, generation, nonce, and the opaque `evidence_hash`; the verifier never derives that hash from the request. An attacker can therefore replace `challenge_started_at` and `challenge_ends_at` with a backdated 24-hour interval and make a not-yet-mature ceremony pass immediately.

- [P1] Compare recovery requests with persisted high-water state — /private/tmp/sonar-agent-dream-outcome/src/truth-contract/trust-control-plane.ts:225-230
  After the trusted state has advanced, an older correctly signed recovery request still passes because these checks compare generation and nonce only with values inside the request. Since the function receives neither the persisted generation nor nonce and returns `ACTIVE`, a generation-2 ceremony can be replayed after generation 5; require and atomically compare the current high-water state.

- [P1] Enforce quorum key validity windows — /private/tmp/sonar-agent-dream-outcome/src/truth-contract/trust-control-plane.ts:180-186
  When a quorum key is not yet valid or its `valid_until` has passed, its signature is still counted because verification never checks either timestamp. This allows expired governance or recovery credentials to authorize rotation/recovery; pass the trusted current time and reject signatures outside each key's interval.

- [P1] Prevent revoked artifacts from being cached green again — /private/tmp/sonar-agent-dream-outcome/src/truth-contract/revocation-control-plane.ts:335-339
  After a revocation is ingested, a later readiness evaluation can call `cacheGreen` for the same artifact and re-add it because the consumer retains no revoked-target set. `isGreen` then returns true for an actively revoked artifact, so caching must refuse targets covered by prior revocations.

- [P1] Evict artifacts affected by key revocations — /private/tmp/sonar-agent-dream-outcome/src/truth-contract/revocation-control-plane.ts:322-324
  For `target_kind: "key"`, this deletes the key ID from a set containing artifact IDs, leaving every artifact signed by the compromised key green. Key revocations must resolve and evict the affected artifact set rather than treating the key ID as an artifact cache key.

- [P1] Bind freshness to the current revocation epoch — /private/tmp/sonar-agent-dream-outcome/src/truth-contract/revocation-control-plane.ts:463-468
  A current checkpoint with revocation epoch 1 can follow a previous checkpoint at epoch 2 and still return `FRESH`, because the epoch is never compared here. This permits revocation-state rollback despite fresh clocks and connected channels; require a nondecreasing epoch and equality with the consumer's trusted current epoch.
