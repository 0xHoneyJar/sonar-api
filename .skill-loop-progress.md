+ 3 PostToolUse); Stop/SessionStart/UserPromptSubmit add more. These are the
p95-latency-defining hot paths of the framework itself. `beads-health.sh` runs
at every workflow boundary. hyperfine is NOT installed — passes use the shared
timing harness committed by pass 1. Tests: `npx bats tests/unit/<file>.bats`,
full suite `tests/run-unit-tests.sh`.

## Zone-guard protocol (passes that edit .claude/hooks/**)
zone-write-guard BLOCKS Write/Edit tool calls to framework zone in this
session (verified empirically; runbook override LOA_ZONE_GUARD_BYPASS=1 is
env-based and cannot be applied to tool calls mid-session). This repo IS
upstream — hook edits are the product. Protocol, per hook change:
1. Author modified hook via Write tool → grimoires/loa/perf/skill-loop-2026-07-05/staging/
2. bash -n; then overwrite the hook's copy INSIDE the bench mirror and prove
   golden parity there FIRST (live session hooks untouched until proven)
3. Atomic install via Bash cp+mv (mutation-logger audits it); then
   golden/capture.sh --verify (all 32 combos) + relevant bats
4. If anything fails: git checkout -- the hook immediately
Deviation is deliberate, operator-invoked, logged here + in final report.
